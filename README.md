## 🛠️ Deep-Dive Subsystem Specifications

### 1. Emulation Frontend (C++ / Qt6)
The user interface handles high-refresh-rate library navigation, rendering complex metadata grids, and embedding real-time gameplay previews.
* **Why C++ & Qt6:** Avoids the heavy memory footprints and garbage collection pauses inherent in Electron or web-tech wrappers. Qt6 provides raw hardware acceleration through Vulkan/OpenGL/Metal backends via the Qt Rendering Hardware Interface (RHI).
* **Core Responsibilities:**
  * Zero-copy presentation of preview video buffers generated by the scraper daemon.
  * Capturing keyboard/controller inputs natively via `evdev` (Linux) or `Raw Input` (Windows) to prevent input lag from leaking into launched emulators.
  * Maintaining a responsive 144Hz+ rendering loop during deep recursive directory scans.

### 2. Gameplay Scraper & Previewing System (Python)
An asynchronous pipeline that monitors library additions, scrapes video clips from online repositories/YouTube, and optimizes them into low-latency local loop previews.
* **Why Python:** Python is the native environment for `yt-dlp`. Scraping relies heavily on string manipulation, network I/O, and handling upstream layout changes from web scrapers. Speed of iteration outweighs raw computation here.
* **Core Responsibilities:**
  * Orchestrating `yt-dlp` instances via `asyncio` to fetch high-quality, timestamp-targeted gameplay demonstrations.
  * Driving `FFmpeg` sub-processes to slice, strip audio tracks, compress, and output hardware-accelerated `.mp4` / `.webm` loops optimized for the Qt6 UI container.
  * Fallback parsing of community-driven scraping APIs with aggressive rate-limiting and connection-pooling protections.

### 3. Emulator-Connectors (Go)
The abstractions responsible for configuration injection, state tracking, and process control over external emulation engines (e.g., DuckStation, PCSX2, RPCS3, Dolphin, Yuzu, RetroArch).
* **Why Go:** Launching, monitoring, and killing system processes requires clean concurrent control blocks. Go’s goroutines, structural select statements, and robust standard library simplify tracking asynchronous process lifecycles.
* **Core Responsibilities:**
  * Translating standard metadata into emulator-specific CLI flags or configuration files dynamically at launch.
  * Monitoring child process PIDs, tracking execution runtime metrics, and detecting unexpected memory faults or crashes without jeopardizing the parent frontend process.
  * Managing runtime memory hacking pipelines (such as reading active emulation states for Discord Rich Presence integrations).

### 4. ROM Search DB & Download engine (Rust)
The storage and retrieval engine capable of executing instantaneous full-text searches across tens of thousands of files, matching CRC32/SHA-1 checksums, and parsing fuzzy titles.
* **Why Rust:** Database queries must resolve instantly (<2ms) to ensure interactive, keystroke-by-keystroke UI filtering. Rust guarantees memory safety while allowing low-level structural packing of indices and SIMD-accelerated fuzzy string matching.
* **Core Responsibilities:**
  * Running an embedded SQLite instance configured with `WAL` (Write-Ahead Logging) mode and custom FTS5 extensions.
  * Executing lightning-fast reverse-index lookups to match raw file hashes against standard No-Intro or Redump databases.
  * Providing thread-safe read access to the Frontend UI while allowing background scrape writes to execute simultaneously without lock contention.

## 🔄 Inter-Process Communication (IPC) Protocol
To maintain decoupled architecture, the modules communicate using an event-driven **Unix Domain Socket / Named Pipe** mechanism utilizing serialized Protocol Buffers (Protobuf). This guarantees strict type safety across C++, Go, Python, and Rust while preserving structural speed.
